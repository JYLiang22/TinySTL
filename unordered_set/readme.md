# 目的

本题需要设计一个 unordered_set 类，实现如下功能：

1. 基础功能

    - 构造函数：初始化 unordered_set 实例
    - 析构函数：清理资源，确保无内存泄露

2. 核心功能

    - 在 unordered_set 中插入一个元素
    - 在 unordered_set 中删除一个元素
    - 判断一个元素是否在 unordered_set 中
    - 判断 unordered_set 是否为空
    - 获取 unordered_set 的大小


# 测试文件介绍


## 输入描述

题目的包含多行输入，第一行为正整数 N, 代表后续有 N 行命令序列。<br>
接下来 N 行，每行包含一个命令，命令格式为 [operation] [parameters] ，具体命令如下：<br>

```
insert 命令：
格式：insert [element]
功能：在 unordered_set 中添加元素，如果元素已经存在，则不进行任何操作

erase 命令：
格式：erase [element]
功能：删除 unordered_set 中的元素，如果元素不存在，则不进行任何操作

find 命令：
格式：find [element]
功能：查询 unordered_set 中的元素

empty 命令：
格式：empty
功能：判断无序集合是否为空

size 命令：
格式：size
功能：获取无序集合的大小
```


## 输出描述

输出为每行命令执行后的结果，具体输出格式如下：<br>

```
insert 命令：无输出
erase 命令：无输出
find 命令：如果元素存在，则输出 true，否则输出 false
empty 命令：如果 unordered_set 为空，则输出 true，否则输出 false
size 命令：输出 unordered_set 的大小
```


# 与标准库的区别

这份代码是一个极简化的模拟，而不是一个准确的 std::unordered_set 的实现。以下是这份代码与标准库中的 std::unordered_set 的一些主要区别：

1. 功能完整性：

    - 标准库的 std::unordered_set 提供了完整的功能集，包括迭代器支持、元素访问、桶接口、负载系数控制、哈希策略和相等比较策略的自定义等。
    - 我们的 Unordered_set 实现看起来相对简单，主要提供插入、删除、查找和清空操作。

2. 迭代器支持：

    - std::unordered_set 提供了迭代器，使得用户可以遍历集合中的所有元素。
    - 我们的 Unordered_set 实现中没有提到迭代器的支持。

3. 异常安全性：

    - std::unordered_set 的成员函数通常提供强异常安全保证。
    - 我们的实现中没有提到任何关于异常处理的代码，可能不提供相同级别的异常安全保证。

4. 类型限制：

    - std::unordered_set 要求键类型必须可哈希和可比较相等。
    - 我们的实现似乎也有这样的要求，但是没有明确说明。

5. 内存管理：

    - std::unordered_set 会对内存管理进行优化，例如，当元素被删除后，可能会进行合理的内存回收或重用。
    - 我们的 Unordered_set 实现中没有提到任何内存管理策略。

6. 性能：

    - 标准库的 std::unordered_set 经过了优化，以提供尽可能好的性能。
    - 我们的 Unordered_set 类性能可能不如标准库版本，取决于 HashTable 实现的细节。
 
7. 方法命名：

    - 标准库中会使用标准的命名约定，例如 find 方法返回一个迭代器而不是布尔值。
    - 我们的实现中 find 方法返回一个布尔值，这与标准库的命名和返回类型约定不一致。

8. 可移植性：

    - std::unordered_set 是标准库的一部分，其行为在所有支持标准库的平台上是一致的。
    - 我们的 Unordered_set 类可能不具备相同的可移植性。

9. 构造函数和析构函数：

    - std::unordered_set 提供了多种构造函数来支持不同的初始化方式，包括复制构造、移动构造等。
    - 我们的实现只提供了默认构造和默认析构函数。

10. 拷贝控制：

    - 标准库的 std::unordered_set 定义了拷贝构造函数、移动构造函数、拷贝赋值运算符和移动赋值运算符。
    - 我们的类没有明确定义这些拷贝控制操作，这意味着如果 HashTable 类没有正确处理它们，可能导致问题。

如果我们的 Unordered_set 类是为了学习或特定环境下的简化使用，那么它可能足够了。但是，如果您需要一个完整且健壮的通用容器，使用标准库的 std::unordered_set 会更加可靠和高效。
